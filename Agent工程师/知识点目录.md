# 一、LLM 推理与生成（Day1-7）

## 1.1 Transformer 架构

- [✅🔴] **Attention 机制** — 核心计算：Q·K^T·V、Softmax 归一化
- [✅🔴] **Multi-Head Attention（MHA）** — 多头并行、拼接投影
- [✅🟠] **Multi-Query Attention（MQA）** — 共享 KV 头、推理加速
- [✅🔴] **Grouped Query Attention（GQA）** — MHA 与 MQA 的折中、分组共享
- [✅🟠] **MLA（Multi-head Latent Attention）** — DeepSeek 的隐空间压缩注意力
- [✅🔴] **RoPE（旋转位置编码）** — 相对位置编码、旋转矩阵实现
- [✅🟠] **FlashAttention** — IO 感知、Tiling、Online Softmax、重计算
- [🟡🟢] **ALiBi / NoPE** — 其他位置编码方案

## 1.2 Decoding 策略

- [✅🔴] **Greedy Decoding** — 贪心搜索
- [✅🟠] **Beam Search** — 束搜索、宽度与多样性
- [✅🔴] **Top-K 采样** — 限制候选数量
- [✅🔴] **Top-P（Nucleus）采样** — 累积概率阈值
- [✅🔴] **Temperature** — 概率分布软化/硬化
- [✅🟠] **Speculative Decoding** — 小模型猜→大模型验、草稿模型

## 1.3 Tokenization

- [✅🔴] **BPE（Byte-Pair Encoding）** — 字节对编码、合并规则
- [🟡🟢] **SentencePiece / Unigram / WordPiece** — 其他分词方案对比

## 1.4 推理优化（系统层）

- [✅🔴] **KV Cache** — 键值缓存、显存 trade-off、序列长度影响
- [✅🔴] **PagedAttention** — 分页管理 KV Cache、非连续存储
- [✅🟠] **Continuous Batching** — 动态批处理、iteration-level 调度
- [✅🔴] **vLLM** — 推理引擎、PagedAttention 实现、生产部署
- [🟡🟢] **Prefix Caching** — 共享前缀缓存
- [🟡🟢] **FlashAttention-2/3** — 进一步优化

# 四、RAG 检索增强（Day8-13）

## 4.1 RAG 基础

- [✅🔴] **RAG 三阶段** — Index（索引）→ Retrieve（检索）→ Generate（生成）
- [✅🔴] **Embedding 模型选型** — BGE / text-embedding-3 / MTEB 基准
- [✅🔴] **Lost in the Middle** — 中间信息衰减、分段策略

## 4.2 Chunking 策略

- [✅🔴] **固定长度分块** — 简单但边界截断严重
- [✅🔴] **语义分块** — Embedding 相似度断崖检测
- [✅🔴] **Overlap（块重叠）** — 相邻块共享上下文
- [✅🟠] **递归字符分块** — RecursiveCharacterTextSplitter
- [🟡🟠] **基于文档结构分块** — Markdown 标题/Markdown 分割
- [🟡🟠] **Agentic Chunking** — LLM 自主决定分块边界

## 4.3 检索策略

- [✅🔴] **稠密检索（Dense Retrieval）** — Embedding 向量相似度搜索
- [✅🔴] **稀疏检索（Sparse Retrieval）** — BM25、TF-IDF 关键词匹配
- [✅🔴] **Hybrid Search（混合搜索）** — 稠密+稀疏互补
- [✅🔴] **RRF（Reciprocal Rank Fusion）** — 多路排序融合算法
- [✅🟠] **查询改写（Query Rewrite）** — 模糊→清晰、LLM 重写
- [✅🟠] **查询分解（Query Decomposition）** — 复合问题拆分子问题
- [🟡🟠] **HyDE（假设文档嵌入）** — 先生成假想答案再检索

## 4.4 Re-ranking 精排

- [✅🔴] **两阶段检索架构** — 粗筛→精排
- [✅🔴] **Bi-Encoder** — 双塔编码、独立编码快但不精
- [✅🔴] **Cross-Encoder** — 交叉编码、准但慢、只能做精排
- [✅🔴] **Bi vs Cross 取舍** — 粗筛用 Bi、精排用 Cross
- [✅🟠] **ColBERT（后期交互）** — 折中方案
- [🟡🟠] **Cohere Rerank / BGE Reranker** — 商业与开源方案

## 4.5 检索指标

- [✅🔴] **Hit Rate（命中率）** — 最相关文档在前 K 中出现了吗
- [✅🔴] **MRR（Mean Reciprocal Rank）** — 最相关文档排第几
- [✅🟠] **NDCG** — 多级相关性排序评估
- [✅🟠] **Precision / Recall @K** — 精确率与召回率

## 4.6 Agentic RAG

- [✅🔴] **Corrective RAG** — 检索→评估→改写→重搜
- [✅🔴] **Self-RAG** — 每条 chunk 自评 isRel/isSup
- [✅🟠] **HyDE（查询侧优化）** — 假想文档增强检索
- [✅🔴] **Multi-Hop** — 跨多轮 ReAct、逐层深入
- [✅🟠] **GraphRAG** — 实体关系抽取、知识图谱增强检索
- [🟡🔴] **Agentic RAG 工具架构** — 独立工具 vs 内部集成策略

## 4.7 向量数据库

- [✅🔴] **ANN 算法** — 近似最近邻搜索
- [✅🔴] **HNSW** — 多层图、精度最高但吃内存
- [✅🔴] **IVF** — 聚类分组、省内存但精度略低
- [✅🟠] **DiskANN** — SSD 索引、十亿级规模
- [✅🔴] **Milvus** — 最成熟分布式、架构重
- [✅🔴] **Qdrant** — Rust 实现、单机性能好
- [✅🟠] **Chroma** — 轻量原型、pip install
- [✅🔴] **PGVector** — PostgreSQL 扩展、零运维
- [✅🟠] **Pinecone** — 纯托管

## 4.8 RAG 评估与生产化

- [✅🔴] **检索质量** — Hit Rate / MRR
- [✅🔴] **生成质量** — 忠实度 / 完整性 / 有用性
- [✅🟠] **LLM-as-a-Judge** — 强模型打分评估
- [✅🟠] **索引漂移** — 数据量增长后参数失效
- [✅🟠] **查询分布漂移** — 测试集与线上不一致
- [✅🟠] **数据污染** — 过时/重复文档、质量分过滤
- [✅🟠] **评估噪音** — 固定测试集、扩大样本量
- [❌🔴] **RAGAS 评估框架** — Faithfulness / Answer Relevancy / Context Precision
- [❌🟠] **长上下文 RAG** — 塞全文 vs 检索折中
