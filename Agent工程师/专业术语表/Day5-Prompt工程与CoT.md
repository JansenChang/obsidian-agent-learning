# 专业术语表 — Day5

## C

**CoT (Chain-of-Thought)**
- 中文：思维链
- 定义：引导模型在输出最终答案前先输出中间推理步骤，显著提升复杂推理任务的准确率
- 首次出现：Day5

## F

**Few-shot (少样本提示)**
- 中文：少样本提示
- 定义：在 prompt 中给 1 到几个示例（含推理步骤），让模型模仿示例的模式回答
- 首次出现：Day5

## P

**Prompt Engineering**
- 中文：提示工程
- 定义：通过设计输入提示词的结构、内容和形式来引导模型输出期望结果的技术
- 首次出现：Day5

## R

**ReAct (Reasoning + Acting)**
- 中文：推理+行动框架
- 定义：将推理步骤（Thought）和行动（Action/Observation）交替结合的 Agent 框架，CoT 只有推理，ReAct 可以执行工具调用
- 首次出现：Day5

## S

**Self-Consistency**
- 中文：自一致性
- 定义：对同一输入多次采样生成多条推理路径，投票选最一致的最终答案，能显著提升准确率
- 首次出现：Day5

## T

**ToT (Tree-of-Thought)**
- 中文：思维树
- 定义：在推理过程中维护多条并行推理分支，必要时回溯剪枝，像搜索树一样探索多种可能性
- 首次出现：Day5

## Z

**Zero-shot CoT**
- 中文：零样本思维链
- 定义：不加示例，只在 prompt 末尾加「请一步步思考」，模型自动产出推理步骤
- 首次出现：Day5
