# 接地（Grounding）

> 一句话：工具调用将 LLM 的推理从"漂浮的猜测"拉回"外部事实地面"，防止幻觉无边界放大。

## 核心机制

### 问题：CoT 的推理为什么会"飘"

LLM 在纯推理（CoT/Reason Only）模式下，每一轮推理基于上一轮的结论：
- LLM 从自身知识中掏出一个"事实"
- 基于该"事实"推导出下一个结论
- 循环往复，形成封闭的自循环

**这整个过程缺少外部验证机制**。如果第一步的"事实"就是错的，后面全部的推理链都是在放大错误。

### 解法：工具作为"真相锚点"

ReAct 的 Action→Observation 环节充当了**外部事实注入点**：

```
LLM 推理："东京酒店大约 500-800 元/晚"
    ↓ Action: 调用酒店 API
    ↓ Observation: API 返回 1200 元/晚
    ↓ LLM 推理（被纠正）："我的预估偏低了，实际价格是 1200，应调整预算"
```

工具返回的**硬数据**与 LLM 的**软推理**产生冲突时，硬数据获胜。这就是"接地"——推理漂浮在空中，工具调用把它拉回地面。

### 为什么接地如此关键

| 无接地（CoT） | 有接地（ReAct） |
|:---|:---|
| 推理链封闭，错误无反馈 | 每轮 Action 都是事实注入 |
| 幻觉在链上逐级放大 | 工具返回值打断错误推理 |
| 用户难以辨别真假 | 真实数据提供可验证性 |

## 面试要点

- **"接地"这个词本身的来源**：Grounding in AI，指符号系统与物理世界的连接。在 ReAct 语境下，工具调用 = 把推理 grounded 到外部世界。
- **面试官追问"什么时候不需要接地"**：当问题完全在 LLM 内部知识范围内（如百科问答），CoT 就够了。强行接地反而引入工具噪音。
- **接地与幻觉的关系**：接地不是消灭幻觉，而是**在幻觉传播链上安装检查点**。每一轮 Observation 都是一次事实核查机会。
