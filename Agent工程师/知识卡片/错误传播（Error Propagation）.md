# 错误传播（Error Propagation）

> 一句话：CoT 自循环推理中，第一步推理错误在无外部验证的封闭链中被逐级放大，最终输出看似合理但完全错误的答案。

## 核心机制

### 错误传播的链条

CoT（Reason Only）模式下，LLM 的推理是**自循环**的：

```
LLM 内部知识 → 推理出"事实 A"（❌ 可能是错的）
    → 基于"事实 A"推导出"事实 B"（❌ 在错误基础上继续）
    → 基于"事实 B"推导出"事实 C"（❌ 进一步放大）
    → ... 最终答案完全偏离真实
```

**关键问题**：LLM 没有内部机制判断自己的第一步推理是否正确。而且由于 LLM 天然倾向于生成流畅连贯的文本，错误推理链读起来非常"合理"，用户更容易被说服。

### 为什么它特别危险

1. **隐蔽性强**：CoT 输出完整的推理过程，看似有理有据，用户难以察觉第一步就错了
2. **放大效应**：不是线性叠加，而是**复合放大**——每步推理都引入了新的偏差
3. **LLM 不会说"不知道"**：当 LLM 缺乏相关内部知识时，它不会承认，而是推理出"看起来合理但完全错误"的答案

### ReAct 如何打断错误传播

ReAct 在推理链中插入外部事实检查点：

```
LLM 推理 → Action（查工具）→ Observation（工具返回真实数据）
    → 如果推理错误，LLM 会在新 Thought 中自我纠正
```

每一次工具调用都是一次**外部验证**，阻断错误在推理链上的传播。

## 面试要点

- **"为什么 CoT 会有幻觉而 ReAct 更少"**：CoT 是封闭推理，错误传播无法自检；ReAct 用外部工具打断错误链
- **"错误传播是否可能发生在 ReAct 中"**：会。如果 LLM 在 Thought 中推理错 → 选了错误的工具 → 工具返回无关信息 → LLM 基于无关信息继续推理 → 偏离更远。这是 ReAct 的第一种失败模式（推理幻觉传播）
- **解法追问**：可在 System Prompt 中加"刹车提示"——"请先检查已有信息是否能回答用户问题，如果能直接回答，不要调用额外工具"
