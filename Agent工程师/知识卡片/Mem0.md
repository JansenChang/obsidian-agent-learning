# Mem0

> 一句话：Mem0 是当前最受关注的长期记忆系统，核心设计解决三个工程问题——自动从对话中提取事实、去重合并冲突信息、按重要性层级化存储。面试中引用 Mem0 的设计逻辑能展示你对工程化记忆系统的深度理解。

## 核心机制

### 三大核心设计

```
问题 1：自动提取
  ├─ 不能指望用户说"请记住我喜欢蓝色"——用户不会主动标记该记什么
  ├─ 方案：专用 LLM 读取最近几轮对话，输出结构化事实列表
  ├─ 每条事实包含：content（内容）、type（偏好/背景/决定/事件）、importance（1-10）
  └─ 这是"从非结构化对话中结构化信息抽取"的经典 NLU 问题

问题 2：去重合并（Memory Merging）
  ├─ 用户第一次说"我住在北京"，第二次说"我搬到上海了"
  ├─ 不能存两条矛盾记录——新信息应覆盖旧信息
  ├─ 方案：每次提取新事实时，先在已有记忆中检索相似内容
  ├─ 发现冲突或更新 → 自动用新的替换旧的（而非追加）
  └─ 这要求向量检索 + 语义比对 + 冲突判断三步联动

问题 3：层级化存储（Memory Layers）
  ├─ 不是所有记忆重要性相同——"讨厌香菜"比"昨天喝了拿铁"重要得多
  ├─ 偏好类 > 背景类 > 决策类 > 事件类
  ├─ 检索时优先召回高层级记忆
  └─ Memory Synthesis 高阶概念：当某话题记忆超过阈值，自动提炼元记忆
      （如 5 次提到"简洁设计"→ 总结"该用户偏好简洁设计风格"）
```

### 记忆注入的时机设计

```
对话开始：全量检索 → 高相关记忆注入 System Prompt
对话中途方向变化：增量检索 → 补充新主题记忆
ReAct 关键决策点（写操作前）：最后一次记忆校验 → 确保不违背用户约束
```

### 实际案例：智能健身教练 Agent

```
需要记住两件事：
  1. 训练数据："上次深蹲 100kg × 5 × 5"——精确数值，下次训练自动建议加重
  2. 伤病约束："三个月前受过膝伤"——高优先级约束，训练计划需避开高冲击动作

ReAct 循环开始时：
  → 检索最近的训练数据 + 伤病约束
  → 注入 Thought："用户上次深蹲 100kg，膝伤需避免高冲击，今日建议：..."
  → 再开始推理训练方案
```

## 面试要点

1. **"自动提取不是魔法"**——主动解释用 LLM 做事实提取的技术方案，而非只说"AI 自动记住"
2. **去重合并是区分点**——能讲清楚"新事实覆盖旧事实"的实现逻辑（检索→比对→替换），不是简单的 CRUD
3. **层级存储讲案例**——用健身教练案例把"两层记忆（数值 + 约束）"说清楚
4. **Memory Synthesis 加分项**——提到"从多条记忆自动提炼元记忆"，展现对产品设计意图的理解
5. **与遗忘策略联动**——Mem0 的去重合并是防冗余，遗忘策略是防膨胀，两者互补

## 关联概念

- 长期记忆（用户专属RAG）：Mem0 是长期记忆的工程化实现方案之一
- 遗忘策略：Mem0 的去重合并与遗忘策略的重要性评分互补
- 记忆更新时机：Mem0 默认采用异步批量更新
- Zep / Letta：同赛道的其他记忆系统，Zep 也有 Memory Synthesis 概念
