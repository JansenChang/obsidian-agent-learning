# 遗忘策略

> 一句话：长期记忆会无限膨胀——一年几万条记忆导致检索效率下降、存储成本增长、旧记忆过时。遗忘策略通过四道防线（时间衰减、重要性评分、容量淘汰、用户主动删除）控制记忆规模，类比操作系统的内存页面置换算法。

## 核心机制

### 四道防线

```
防线 1：时间衰减
  ├─ 旧事件降权（三个月前的某次对话细节大概率无价值）
  ├─ 但核心偏好不过期（"我对花生过敏"一年前说的，绝不能删）
  └─ 实现：给每条记忆打时效性标签，偏好类永久，事件类随时间衰减

防线 2：重要性评分
  ├─ 初始分：LLM 提取时赋予（偏好 > 背景 > 事件）
  ├─ 动态加减：高频被检索 +1，从未被检索 -1
  ├─ 阈值淘汰：降到阈值 → 自动归档冷存储或删除
  └─ 实现：每条记忆维护一个 score 字段，定期巡检

防线 3：容量淘汰
  ├─ 当向量库存储量 > 设定上限 → 触发淘汰
  ├─ 按重要性分数从低到高驱逐（完全类比 OS 页面置换 LRU/LFU）
  └─ 实现：低分记忆先归档（S3/OSS 冷存储），再超时后彻底删除

防线 4：用户主动删除
  ├─ GDPR / 隐私法规要求：用户有权要求忘记
  ├─ 按用户 ID 精确删除所有相关记忆
  ├─ 注意事项：已注入历史的摘要无法撤回，但确保新对话不再检索到
  └─ 实现：数据库级联删除 + 审计日志记录
```

### 关键设计原则

| 原则 | 说明 |
|------|------|
| **遗忘 ≠ 简单删除** | 先归档冷存储，确认无价值再真删 |
| **时间衰减不等于均匀衰减** | 偏好信息（过敏史、管理权限）永不过期，事件信息（昨天喝了什么）快速衰减 |
| **OS 页面置换类比** | 面试中说出这个类比，展示跨领域工程思维的迁移能力 |

## 面试要点

1. **"为什么需要遗忘"**——先说问题（无限膨胀），再说四道防线，逻辑链条清晰
2. **时间衰减的例外**——核心偏好不受时间影响，这是区分好坏设计的关键细节
3. **重要性评分的动态性**——不是静态分数，而是随检索频率动态变化
4. **OS 页面置换类比**——容量淘汰和内存管理思想完全一致
5. **用户删除的审计**——不仅是删数据，还要有审计日志，这是合规要求
