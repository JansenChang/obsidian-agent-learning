# 系统提示词分层设计

> 一句话：将 System Prompt 按优先级倒金字塔拆为四层——角色→任务→约束→知识，其中约束层优先级最高（安全压倒一切），前三层作为静态前缀可被 KV Cache 复用，知识层可动态注入。

## 核心机制

### 四层倒金字塔

```
         ┌──────────────┐
         │  约束（最高）  │  "不能修改订单" "不能泄露 PII"
         │  ⚠️ 不可违背   │  ← 安全规则，所有层中优先级最高
         ├──────────────┤
         │    任务        │  "查询订单/跟踪物流/解答政策"
         │  能力范围定义   │  ← 告诉 LLM 能调用什么工具
         ├──────────────┤
         │    角色        │  "你是一个智能订单客服助手"
         │  身份与语气     │  ← 决定回答风格和定位
         ├──────────────┤
         │  知识（最低）   │  "品牌信息/退换货政策/物流合作方"
         │  可动态注入     │  ← RAG 检索结果、用户记忆、动态配置
         └──────────────┘
```

**倒金字塔含义：** 越靠上的层优先级越高。约束 > 任务 > 角色 > 知识。这意味着即使知识层包含了与约束冲突的信息，约束层也必须胜出。

### 分层核心原则

| 原则 | 说明 | 面试表达 |
|------|------|---------|
| **安全压倒一切** | 约束层的指令永远不能被下层覆盖 | "约束是硬边界，不是软建议" |
| **不变在前** | 角色+任务+约束 = 静态前缀，可缓存 | "前三层是 KV Cache 的核心资产" |
| **动态在后** | 知识层可每请求不同（RAG 结果/用户记忆） | "知识注入是动态后缀，不破坏前缀缓存" |
| **层间不交叉** | 一层只做一件事，不要角色定义里夹约束 | "分层清晰 → 维护成本低 → 安全可审计" |

### 与 ReAct 提示词四层设计的关系

| Day18 系统提示词分层 | Day15 ReAct 提示词四层 |
|---------------------|------------------------|
| 角色 | 角色定义 |
| 任务 | 推理行动约束 |
| 约束 | 安全边界 |
| 知识 | 工具使用指南（可动态注入） |

本质是同源设计，但 Day18 强调了 KV Cache 优化维度和倒金字塔优先级。

## 面试要点

1. **"倒金字塔"是面试金句**——面试官问"怎么设计 System Prompt"，你说"四层倒金字塔：约束 > 任务 > 角色 > 知识"，三秒钟建立架构感
2. **为什么约束优先级最高？**——因为 System Prompt 是 LLM 的行为宪法，安全约束必须凌驾于所有其他指令之上
3. **层间冲突的处理**——约束层 vs 知识层冲突 → 约束胜出；角色层 vs 任务层冲突 → 任务胜出（不要让"语气友好"覆盖"不能做的事"）
4. **知识层的动态注入不破坏前缀缓存**——这是分层设计的工程价值：前三层不变 = KV Cache 命中，知识层可变 = 每用户差异化
5. **与 Anthropic System Prompt 实践一致**——Anthropic 官方推荐 System Prompt 结构：能力声明 → 约束 → 格式要求 → 知识注入，本质也是分层思想

## 关联概念

- Write策略：系统提示词分层是 Write 策略中 System Prompt 结构化写入的实现
- 前缀缓存与静态前缀分离：前3层不变 = 静态前缀可缓存，第4层动态 = 差异化后缀
- ReAct提示词四层设计：Day15 的同源设计，关注推理-行动约束
- Context Engineering四大策略：系统提示词分层是 Context Engineering 在 Prompt 设计层面的核心实践
