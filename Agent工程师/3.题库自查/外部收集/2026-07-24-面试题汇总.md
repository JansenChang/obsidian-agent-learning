## 一、Agent 基础（10题）

### 1.1 概念与本质

**Q1:** 请简述 Agent 是什么？和普通 Chatbot 有什么区别？
- **难度:** 🔴 必考
- **来源:** B站:图灵FOX老师(BV1L1VJ6nEei)
- **答案要点:** Agent = LLM + 工具使用 + 记忆 + 规划 + 行动。Chatbot 只做一问一答，Agent 能自主感知环境、制定计划、调用工具、循环执行直到目标达成。

**Q2:** 为什么 2026 年所有人都在入局 Agent 开发？

- **来源:** B站:图灵FOX老师(BV1L1VJ6nEei)

**Q3:** Agent 开发的核心就是上下文工程（Context Engineering），你认同吗？

## 二、ReAct / 工具调用（8题）

### 2.1 Function Calling

**Q11:** Function Calling 的底层原理是什么？厂商在后台用什么技术保证结构化输出？
- **难度:** 🔴 必考
- **来源:** B站:高德面经(BV1Gk766mETp) + GitHub:LLMInterviewQuestions
- **答案要点:** 模型被训练输出特殊token标记的函数调用格式; constrained decoding(受限解码)在推理时强制token序列符合schema; grammar-based sampling保证JSON格式合法。

**Q12:** 设计工具调用时，如何防止工具超载（给 Agent 传了 100 个工具描述）？
- **难度:** 🟠 加分
- **来源:** B站:吴恩达大模型(BV1v9Kg6BEEe)
- **答案要点:** 动态工具检索（只传相关的5-10个）；工具分组分层；按场景预筛选。

**Q13:** 工具调用时，模型容易忘记参数（如发件人），如何从工程侧解决？
- **难度:** 🟠 加分
- **来源:** B站:高德面经(BV1Gk766mETp)

**Q14:** Agent 调用的 API 返回报错了怎么办？如何设计重试和纠错机制？
- **难度:** 🔴 必考
- **来源:** B站:高德面经(BV1Gk766mETp)
- **答案要点:** 指数退避重试 + 错误信息注入Observation让LLM自行判断 + 最大重试次数 + 降级策略。

### 2.2 结构化输出

**Q15:** 要保证 Agent 稳定输出结构化数据，底层架构有哪些核心兜底手段？

## 四、LLM 推理（10题）

### 4.1 Transformer 原理

**Q26:** 请简述 Transformer 架构的设计理念，以及它为什么能取代 RNN/LSTM？
- **难度:** 🔴 必考
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe) + GitHub:多个
- **答案要点:** 自注意力机制并行计算 vs RNN串行; 直接建模任意距离依赖; 训练效率高。QKV：Q查询/K键/V值。

**Q27:** Transformer 的 QKV（自注意力机制）是如何工作的？
- **难度:** 🔴 必考
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe)

**Q28:** Transformer 的三大工程落地难点是什么？
- **难度:** 🟠 加分
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe)
- **答案要点:** 显存爆炸（KV Cache随序列长度增长）、长度外推性差（训练长度外效果下降）、推理速度瓶颈（逐token自回归）。

### 4.2 推理优化

**Q29:** Flash Attention 解决了什么问题？原理是什么？
- **难度:** 🔴 必考
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe) + GitHub
- **答案要点:** 解决Attention显存带宽瓶颈。Tiling分块计算（SRAM内完成）+ Online Softmax + Recomputation(反向重算不存中间矩阵)。

**Q30:** RoPE（旋转位置编码）如何解决长度外推性问题？
- **难度:** 🟠 加分
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe)

**Q31:** MQA / GQA 是什么？它们如何优化推理吞吐量？
- **难度:** 🔴 必考
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe)
- **答案要点:** MQA=所有头共享KV → 极致省显存。GQA=分组共享KV → 折中方案。KV Cache减到1/头数或1/组数。

**Q32:** Encoder-Decoder 架构 vs Decoder-Only 架构各适合什么场景？
- **难度:** 🟠 加分
- **来源:** B站:AI大模型-阿水(BV15kRsBAEYe)
