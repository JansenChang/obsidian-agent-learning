# ANN 算法速查 — HNSW / IVF

> 📌 Day 9 预学，待 Day 12 Vector DB 选型时回来看。

## 为什么需要 ANN

精确搜索 = 问题向量 vs 所有文档向量逐条比 → O(n)，百万级太慢。
ANN = 只跟最可能相关的一小撮比 → O(log n)，1% 精度换 10000 倍速度。

## IVF（Inverted File Index，倒排文件索引）

先聚类分成 1000 个群 → 问题向量跟 1000 个中心点比 → 进最近 3 个群 → 群内精确搜。

**锚点：📚 图书馆分类法** — 先走到「计算机→Python」书架，再逐本翻。

**坑：** 边界文档可能落在隔壁群搜不到。

## HNSW（Hierarchical Navigable Small World，分层可导航小世界图）

三层图结构：顶层稀疏（少数枢纽） → 中层加密 → 底层全量密集连接。
搜索：从顶层开始 → 跳 → 降层 → 跳 → 再降 → 底层精细搜。

**锚点：🏢 高层电梯 + 楼层步行** — 先坐电梯到大致楼层，再步行穿过走廊，最后逐间确认。

**vs IVF：** 无边界问题，精度更高，但建索引更慢。

## 面试一句话

> 「ANN 的核心权衡是用轻微精度损失换巨大速度提升。HNSW 是当前生产中最常用的算法，分层图结构让它同时兼顾精度和速度。」
