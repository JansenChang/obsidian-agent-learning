## ⚡ 面试模拟：完整回答框架

> **「设计一个多轮对话 Agent 时怎么处理上下文？」**

| 层次        | 你的回答                                      |
| --------- | ----------------------------------------- |
| **底层架构**  | 选 GQA 架构模型（LLaMA/DeepSeek），降低 KV Cache 成本 |
| **位置编码**  | RoPE 支持更长上下文，外推性好                         |
| **推理加速**  | KV Cache + FlashAttention + vLLM 部署       |
| **上下文管理** | Lost in the Middle → 系统指令开头、最新结果结尾、中间摘要   |
| **解码控制**  | 客服温度 0.3，创意 0.8，Top-P 0.9                 |
| **安全防护**  | 权限隔离 + 输入输出过滤 + Guardrails                |

> 从底层架构到上层应用全覆盖，体现系统性技术理解力。

## 🔗 交叉连接速查

| 连接 # | 主题 A × 主题 B | 关系 |
|:---:|------|------|
| 1 | Attention × KV Cache | Attention 计算依赖 K/V，缓存就是缓存这些 |
| 2 | Decoding × KV Cache | 每步解码追加新 token 的 K/V 到缓存 |
| 3 | GQA × KV Cache | GQA 减少 KV 头数 → 缓存缩小 4× |
| 4 | Attention × Prompt Injection | Attention 不区分指令/数据 = 注入根源 |
| 5 | CoT × Decoding | CoT 改变输入分布 → 影响 Decoding 的选择空间 |
| 6 | RAG × Prompt Injection | RAG 抓外部数据 = 间接注入的入口 |
| 7 | Tokenization × KV Cache | 压缩率直接影响序列长度 → 间接影响缓存大小 |
