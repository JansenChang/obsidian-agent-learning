# KV Cache 公式

## 核心记忆公式

```
总 KV Cache = 序列长度 × 隐藏维度 × 层数 × KV 压缩比 × 4
```

| 变量 | 含义 | 去哪找 |
|------|------|--------|
| 序列长度 | 生成了多少个 token | 需求定 |
| 隐藏维度 | 模型有多"宽"（hidden_size） | config.json |
| 层数 | 模型有多"深"（num_hidden_layers） | config.json |
| KV 压缩比 | MHA=1, GQA=1/4, MLA≈1/64 | KV Head / Attn Head |
| ×4 | 固定值：K 和 V 各 2 字节（float16）| — |

## 速查表

| 模型 | d | 层数 | KV 压缩 | 每 token | 4K 序列 | 32K 序列 |
|------|---|---|---------|---------|--------|---------|
| LLaMA-2 7B | 4096 | 32 | MHA=1 | 0.5MB | **2GB** | **16GB** |
| LLaMA-3 8B | 4096 | 32 | GQA=1/4 | 0.125MB | **0.5GB** | **4GB** |
| DeepSeek-V2 | 4096 | 60 | MLA≈1/64 | 15KB | **60MB** | **0.5GB** |

## 并发推算

```
每路 KV Cache = 序列长度 × 隐藏维度 × 层数 × KV 压缩比 × 4
可用显存 = 总显存 - 模型权重 - 系统开销（~2GB）
并发路数 ≈ 可用显存 ÷ 每路 KV Cache
```

**例：A100-80GB + LLaMA-3 8B（16bit）+ 序列 32K**
- 模型权重 ≈ 16GB
- 可用显存 ≈ 80 - 16 - 2 = 62GB
- 每路 KV Cache = 4GB
- 最大并发 ≈ 62 ÷ 4 ≈ **15 路**
