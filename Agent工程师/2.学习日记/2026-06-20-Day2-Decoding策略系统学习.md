# Day 2: Decoding 策略与推理优化

---

## 一、Decoding 在整个推理链路中的位置

```
输入 → Embedding → RoPE → Attention → FFN → ... → 最后一层
                                                        ↓
                                                  LM Head → Logits（5万维向量）
                                                        ↓
                                                  Softmax → 概率分布
                                                        ↓
                                          ╔══════════════════════╗
                                          ║  Decoding 策略登场  ║
                                          ║  从这个分布中选一个  ║
                                          ╚══════════════════════╝
                                                        ↓
                                                    输出 token → KV Cache
```

**Decoding 本质**：模型每步输出一个 5 万维的概率分布，Decoding 策略决定**怎么挑一个 token**。

---

## 二、5 种核心策略

### ① Greedy Decoding（贪婪解码）

**做法**：每次都选概率最高的词。

| 优点 | 缺点 |
|------|------|
| 简单、快、确定 | 重复、无多样性、容易陷入循环 |

**实际定位**：适合翻译、摘要这种确定性场景。不适合对话、创作。

**设计来源**：这是最直观的 baseline——既然有概率分布，那每次都选最安全的。后续策略都是在解决它的不足。

**为什么需要**：没有 Greedy 作为 baseline，无法衡量其他策略改善了哪里。

---

### ② Temperature（温度采样）

**做法**：在 softmax 前把 logits 除以 T，改变概率分布的形状。

```
T=0.1 → 分布极陡峭 → 几乎 one-hot（最确定）
T=0.7 → 正常聊天
T=1.0 → 原始 softmax 分布
T=2.0 → 接近均匀（基本随机）
```

**参数怎么来的**：
- 从**统计物理（Boltzmann 分布）**借来的概念
- 物理中 T 控制粒子处于高能态的概率；这里控制模型"冒险"的概率
- 本质是：**不同场景需要的随机性不同**——写诗需要惊喜，解答数学题需要确定性

**物理意义**：让 softmax 分布可伸缩——不调 T，你永远只有一个固定"创造度"的模型。

---

### ③ Top-K 采样

**做法**：只保留概率最高的 K 个词，其余归零重新归一化。

| 优点 | 缺点 |
|------|------|
| 砍掉尾部噪声词 | K 值是固定的，分布陡峭/平坦时都不合适 |

**参数怎么来的**：Fan et al. 2018 年提出。实验发现词表尾部 99% 的词概率接近零，保留它们就是给模型加噪声。K=40 是经验值，不是公式推出来的。

**为什么需要**：没有 Top-K，极低概率词（日语汉字、数学符号、XML标签）每次都有微小概率被选中，相当于随机注入错误。

---

### ④ Top-P / Nucleus Sampling（核采样）

**做法**：不固定 K，从高到低累加概率，加到 ≥P 就停。保留这部分词重归一化。

```
P=0.9：
  「我」0.3 + 「想」0.2 +「去」0.15 +「吃」0.12 +「饭」0.10 = 0.87
  +「了」0.04 = 0.91 ≥ 0.9 ✓ → 取这 6 个词
|
比 Top-K 多一次排序计算，但候选数动态适配
```

**参数怎么来的**：Holtzman et al. 2019 年提出。他发现 Top-K 在分布陡峭时砍得太少（把噪声放进来），分布平坦时砍得太多（把合理的候选丢了）。P=0.9 是实验验证的通用推荐值。

**与 Top-K 的区别**：

| | Top-K | Top-P |
|---|---|---|
| 候选数 | 固定（K=40） | 动态（根据分布） |
| 分布集中时 | 保留太多噪声 | 只保留几个核心 |
| 分布分散时 | 砍掉合理候选 | 保留较多候选 |
| 谁更好？ | 简单 | 更灵活，当前主流 |

---

### ⑤ Beam Search（束搜索）

**做法**：同时维护 N 条候选路径，每步扩展后综合排序保留 top-N。

| 优点 | 缺点 |
|------|------|
| 全局最优（不贪心） | 慢 N 倍，长序列易重复 |

**参数怎么来的**：2000 年代机器翻译界提出的，不是为 LLM 发明的。

**为什么需要**：Greedy 的局部最优陷阱——当前步选最高概率词，可能把后面更好的组合砍掉了。

## 三、常见工程组合

```
实际生产中用组合，不是单选：
  Temperature × Top-K × Top-P

执行顺序：
  1. Top-K 砍尾部噪声
  2. Top-P 动态裁剪候选
  3. Temperature 调分布形状
  4. 采样选一个
```

| 场景 | 推荐配置 |
|------|---------|
| 代码生成 | T=0.2, P=0.9 |
| 翻译/摘要 | Greedy / T≈0.1 |
| 创意写作 | T=0.8, P=0.95 |
| 客服回答 | T=0.3, P=0.9 |
| 数学推理 | T=0, Greedy |

**工程经验**：大多数场景用 T=0.7 + P=0.9 + K=40 作为安全默认值，然后微调。

## 四、各大模型的 Decoding 差异

| 模型 | 默认配置 | 特点 |
|------|---------|------|
| GPT-4/ChatGPT | T=0.7, P=0.9, K=40 | 均衡通用 |
| Claude 3/4 | T≈0.5~0.7 | 偏低，强调可控和安全 |
| DeepSeek-V3/V4 | T≈0.7, P=0.9 | 类似 GPT，但 Thinking Mode有特殊内部解码 |
| Gemini | T≈0.4~0.6 | 偏低，偏向事实性任务 |
| o1/Reasoning模型 | 特殊内部解码 | 内部有CoT解码，用户暴露的T=1或固定配置 |
| 代码模型 | T=0.2, P=0.9 | 追求确定性 |

**GPT 独有的参数**：
- `frequency_penalty`：根据词在已生成文本的出现频率衰减概率
- `presence_penalty`：只要出现过就扣分（不管几次）
- 这两个不是标准 Decoding 策略，是 OpenAI 在采样后加的惩罚项

## 五、与之前学习内容的关联

### Transformer → Decoding

```
之前学的 Attention：算的是 Q×K^T，输出 Context Vector
现在学的 Decoding：在 softmax 输出上操作，选一个 token
两者是上下游关系：Attention 算得好 → 概率分布准 → Decoding 选得对
```

### FlashAttention → Decoding

FlashAttention 加速 Attention 计算 → 更快拿到概率分布 → Decoding 等得更少。

### MQA/GQA → Decoding

MQA/GQA 优化 KV Cache → 每次 Decoding 读 KV Cache 更快。

### RoPE → Decoding

RoPE 的位置编码外推能力（YaRN），决定了长上下文中概率分布是否准确——编码崩了，Decoding 怎么调也没用。

### Self-Attention vs FFN 分工 → Decoding

```
Attention 负责：token 间关系（语境影响）
FFN 负责：提取词义知识（语义影响）
Decoding 负责：在最终概率分布上选词（策略影响）

三者决定了"模型说了什么"：
  Attention 决定要关注什么上下文
  FFN 决定从训练知识中提什么概念
  Decoding 决定从候选词里选哪个
```

## 六、与贝叶斯模型有关联吗？

**直接没有，但思想上有个有趣的对应关系——不是技术上的，是哲学上的。**

### 为什么说"没直接关联"

- 贝叶斯模型的核心是**先验概率 × 似然 = 后验概率**，是一个连续的概率更新框架
- LLM 的 Decoding 策略是**从已知概率分布中选一个 token**，没有概率更新过程
- 模型已经一次性算出了所有候选词的概率，Decoding 只负责选

### 思想上的关联点

| 贝叶斯概念 | Decoding 中类似的思想 |
|-----------|-------------------|
| **先验分布**（prior） | 模型训练数据中学到的词频分布 |
| **后验概率**（posterior） | softmax 输出——条件于上下文的词概率 |
| **最大后验估计（MAP）** | Greedy Decoding——取概率最高的 |
| **后验采样（Posterior Sampling）** | Top-P + Temperature——从完整分布中采样 |
| **似然函数** | 模型内部的 logits 计算 |
| **贝叶斯模型平均** | Beam Search——多条候选路径综合 |

### 一句总结

> Decoding 策略和贝叶斯模型没有技术实现上的关系。但思想上，Greedy ≈ MAP，采样 ≈ Posterior Sampling，Beam Search ≈ 多假设推断。面试官不会问这个关联，但如果被问到"Greedy 和 MAP 什么关系"可以这么答。
